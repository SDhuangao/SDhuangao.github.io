Android Hexo Blog Set-up

0. Prerequisite

  1. Android Phone with root access (Redmi K20 Pro Premium Edition, 12G RAM, 512G Disk, MIUI 12, Android 10)(关于刷机，也可以再记录一波，先flag一下, and also: shitty iOS !!!)
  2. Full Internet Access (FULL)
  3. Assume you have already deployed hexo in mac or windows (Actually you don’t need this, this is just for transferring my blog sources. NextCloud and File Sync).

1. Android Setup

  1. Install Termux from Google Play, F-Droid or CoolApk, grant it super user permission (in Magisk) and file access permission (rum termux-setup-storage in termux, reference).

  2. Open Termux and input (need full network)

  3. Install Hexo

  4. Some commands: official documentation is always the best

  5. Deploy to github

    Now you need to add the key (a long and tedious code block) to the ssh key section in your Github setting.

    NOTICE: You cannot install hexo on external drive (ex: your normal file system: /storage/emulated/0, reference1 and reference2)

  6. Some Commands

2. File Sync System

  1. It’s always pain in the ass to write and save file in the root system, and it’s kind of dangerous. So consider use FolderSync to sync the markdown files and images from emulated to root (In my case, it’s from /storage/emulated/0/Documents/BLOGS to /data/data/com.termux/files/home/blog), and you can enable the scheduled sync function.
  2. Markdown Editor in Android: Markor and Joplin. Markor supported immediate compiling but Joplin support Nextcloud syncing.
  3. Use FolderSync to TWO-WAY sync sources of blog to the folder of Nextcloud Android(Schedule it). Use Nextcloud to TWO-WAY sync between Mac/Windows and Android. Use freefilesync to sync the original blog sources to the folder of Nextcloud Mac/Windows(Schedule it). So, all done here. Just write and create.
